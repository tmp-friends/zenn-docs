# ブログ執筆ガイドライン

このドキュメントは、Komekamiの文体を再現するためのブログ執筆ガイドラインです。

## 文体・トーンの特徴

### 基本的な語り口
- 丁寧語を基調とした親しみやすい文体
- 「です・ます」調で一貫しているが、堅すぎず読みやすい
- 読者への配慮が感じられる語りかけるような口調
- 専門的な内容でも分かりやすく噛み砕いて説明

### 表現の特徴
- 素直な感想・学びを表現
  - 「〜ということがわかった」
  - 「〜と思った」
  - 「〜が印象的だった」
  - 「理解が深まった」
- 学習・成長への意欲を表現
  - 「次は〜を学んでいきたい」
  - 「今後も〜していきたい」
  - 「〜について調べてみる」

## 記事構成パターン

### 1. 論文メモ系記事
構成テンプレート：
```
## [論文タイトル]
- 論文URL:
- 会議:
- （GitHub: ）

### 読む動機
- 具体的で率直な動機を記述

### 要約
- 3-5行程度で簡潔にまとめる

### 背景
- 問題設定や既存手法の課題

### 提案手法
- 数式やアーキテクチャ図を含めた詳細説明

### 有効性
- 実験結果や性能評価

### 感想
- 率直な感想と今後への展望

### 参考
- 関連リンク
```

### 2. 技術解説・実装系記事
構成テンプレート：
```
## はじめに
- 背景・動機をリスト形式で簡潔に

## 概要・要約
- 記事で扱う内容の全体像

## [メインコンテンツ]
- 段階的な説明
- コードブロックや図表を効果的に活用

## おわりに
- 学びや感想
- 今後の展望や改善点

## 参考
- 参考にした資料
```

### 3. 振り返り・まとめ系記事
構成テンプレート：
```
## はじめに
- 背景・概要説明

## [対象の説明・概要]
- 扱ったものの詳細

## 振り返り・まとめ
### Keep
- 良かった点・継続すべき点

### Problem
- 課題・問題点

### Try
- 改善策・今後の取り組み

## おわりに
- 全体的な感想と今後の展望

## 参考
- 参考資料
```

## 文章表現のガイドライン

### 段落・見出しの使い方
- 見出しは階層を明確に（##, ###, ####を適切に使用）
- 1段落は簡潔に（3-5行程度）
- 箇条書きを積極的に活用（リストが多用されている）

### 専門用語の扱い
- 初出時は説明を併記
- 略語は正式名称→略語の順で記載
- 背景知識が必要な概念は丁寧に説明

### コードや数式の扱い
- コードブロックは言語指定を必須
- 数式は適切にマークアップ
- 図表やスクリーンショットを効果的に使用

### 文体・書式のスタイル
- 強調（Bold）は最小限に留める
  - 過度な強調を避け、自然な読みやすさを重視
  - 見出し語や絶対に必要な箇所のみに使用
- リスト項目はハイフン（-）で統一
  - アスタリスク（*）は使用しない
  - ネストしたリストも含めて一貫性を保つ
- 実務寄りの表現を心がける
  - 「こう使うと便利」「落とし穴 / 罠」「パフォーマンスTips」など実用的な観点

## 記事の始め方・終わり方

### 記事の導入
- **背景・動機を箇条書きで明確に**
- **「〜について学びたい」「〜に興味を持った」など学習動機を率直に**

### 記事の結び
- **「〜ができてよかった」「理解が深まった」など素直な達成感**
- **「今後は〜していきたい」など前向きな今後の展望**
- **「〜が面白かった」「〜を試してみたい」など知的好奇心の表現**

## 具体的な文体例

### 好ましい表現
- 「〜について調べていたので、まとめようと思います」
- 「〜を読んでみると、得心のいくことが多く理解が深まりました」
- 「〜は知っていたのですが、実際に使われていて驚きました」
- 「〜あたりの話がよくわからなかった」
- 「軽くお気持ち実装してみようかなと思いました」

### 避けるべき表現
- 過度にカジュアルな表現
- 断定的すぎる表現（「〜である」より「〜だと思います」）
- 冗長な前置き

## 技術記事特有の工夫

### コード例の書き方
- ファイル名を明記（例：`docker-compose.yml`）
- 変更点はdiff形式で表示
- 実行例・出力例を併記

### 注意点の書き方
- 重要な注意事項は明確に
- つまずきやすいポイントを事前に説明
- 代替案や解決策も併せて提示

## メタデータ（Zenn記事設定）
- title: 簡潔で分かりやすく
- emoji: 記事内容に関連したもの
- type: "tech" (技術記事)
- topics: 関連技術を3-5個程度
- published: true
- publication_name: 組織投稿の場合のみ指定

## 学習・成長マインドセットの表現
記事を通じて以下のような学習への姿勢を表現：
- 継続的な学習への意欲
- 試行錯誤を恐れない姿勢
- 知らないことを素直に認める謙虚さ
- 新しい技術への好奇心
- 実践を通じた学びを重視

このガイドラインに従って、Komekamiらしい技術ブログ記事を作成してください。
